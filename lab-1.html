Для належної роботи WEB-застосунку з продажу, необхідне виконання таких бізнес процесів:

  • додавання нового продукту (адміністратором WEB-застосунку);
  • редагування уже наявного продукту(адміністратором WEB-застосунку);
  • видалення продукту (адміністратором WEB-застосунку);
  • перегляду інформації по продуктах;
  • авторизація користувача та адміністратора (з відокремленням по ролях);
  • оформлення замовлення (авторизованим користувачем);
  • опрацювання замовлення (адміністратором WEB-застосунку).

Згідно бізнес вимогам, WEB-застосунок з продажу пончиків, повинен мати функціонал, який задовольняє усім, з вище описаних, бізнес процесам.

Місце розташування звітного HTML-документу:

  • Гіперпосилання.

  1. Будь-який HTML-документ повинен містити рядок оголошення версії HTML-документу <!DOCTYPE html>

  2. Одразу за оголошенням версії документу іде кореневий тег <html></html> в середині якого знаходяться усі елементи HTML-документу.

    1. За допомогою <head></head> містить метадані заданого HTML-документу.

      1. За допомогою <title></title> задається назва HTML-документу, яка у подальшому відображається у заголовку вкладки браузера.

    2. Тег <body></body> визначає тіло HTML документа, та є контейнером для всього видимого вмісту, такого як заголовки, абзаци, зображення, гіперпосилання, таблиці, списки тощо.

HTML-код з проекту WEB-застосунку:

Відображення HTML-документу в проекті WEB-застосунку:

У Web-застосунку не використовується елемент HTML-таблиці.

Тег <img> використовується для додавання зображення в HTML документ. На прикладі Web-застосунку, HTML-тег <img>, використовується, щоб додати зображення у картку товару.

Тег <img> має 2 обов'язкові атрибути:

  • src - атрибут в якому вказується шлях до зображення, яке необхідно завантажити на сторінку. Шлях може бути як абсолютний, так і відносний;
  • alt - атрибут в якому вказується повідомлення, яке буде показано користувачу, якщо зображення не було завантаженно на сторінку (наприклад в src атрибут було невірно задано шлях до зображення).

Для стилізації зображення можна використовувати CSS стилі. У випадку WEB-застосунку для стилізації зображення картки товару, у тег <img> було добавлено атрибут class із значенням card-img-top з бібліотеки bootstrap.

Тег <ul></ul> використовується для додавання непронумерованого списку в HTML-документ. Для додавання пронумерованого списку використовується HTML-тег <ol></ol>. Для того щоб добавити рядок в список (непронумерований, або пронумерований), необхідно добавити тег <li></li>, як значення тегу <ul></ul>, або <ol></ol>.

HTML-код з проекту WEB-застосунку:

Відображення навігації WEB-застосунку із застосуванням HTML-списку:

Тег <a></a> використовується для створення гіперпосилання, в HTML-документі, для переходу з однієї сторінки WEB-застосунку на іншу. На прикладі Web-застосунку, HTML-тег <a></a>, використовується для перемикання діалогових вікон автентифікації та реєстрації користувача.

Тег <a></a> має 1 обов'язковий атрибут:

  • href - атрибут в якому вказується шлях на який повинен виконатись перехід, у разі натиснення користувачем на задане гіперпосилання. Шлях може бути як абсолютний, так і відносний.

У випадку WEB-застосунку, у тег <a></a> було добавлено атрибут id, який використовується в JS коді для зміни базової логіки роботи елементу. Атрибут href із значенням "#" (так, як у посиланні немає необхідності, тому що логіка роботи елементу буде описана в JS коді), та значення тега <a></a> у вигляті текстового повідомлення "Я не зареєстрований!".

Відображення гіперпосилання в проекті WEB-застосунку:

  1. HTML (англ. HyperText Markup Language — мова розмітки гіпертексту) — стандартизована мова розмітки документів для перегляду веб-сторінок у браузері.

  2. CSS (англ. Cascading Style Sheets — каскадні таблиці стилів) - мова, що використовується для стилевого оформлення HTML (і не тільки) сторінок.

  3. JavaScript (JS) це динамічна, об'єктно-орієнтована прототипна мова програмування. Найчастіше використовується для створення сценаріїв веб-сторінок, що надає можливість на стороні клієнта (пристрої кінцевого користувача) взаємодіяти з користувачем, керувати браузером, асинхронно обмінюватися даними з сервером, змінювати структуру та зовнішній вигляд HTML-документу.

HTML разом із CSS - разом утворюють основу дизайну сучасних WEB-застосунків. А JS, дає можливість WEB-застосункам бути інтерактивними.
